# MobileSlicer 0.1.3 Beta

MobileSlicer 0.1.3 Beta tightens the multi-nozzle release checks introduced in
0.1.2. The app now rejects multi-material slices when the generated G-code does
not carry the expected color, filament map, and nozzle metadata.

This is the recommended beta build for Android testers.

## Highlights

- Strengthens MobileSlicer's generated G-code audit for multi-material output.
- Verifies filament colors, filament mapping, toolchange evidence, and
  multi-nozzle metadata before accepting a multi-slot slice.
- Expands the Snapmaker U1 Orca reference comparator to check nozzle metadata,
  prime tower settings, filament colors, and a minimum prime tower feature
  count.
- Shows the latest beta version, APK name, and SHA-256 on the public download
  page.
- Adds a release verification script for GitHub tag, asset, APK metadata, and
  checksum checks.

## Install

Download `MobileSlicer-0.1.3-beta-v6.apk` from the assets below, then open it on
your Android device. Android may ask you to allow installs from your browser or
file manager before continuing.

If you already have an older MobileSlicer beta installed, this build should
install as an update.

## Verified

- Android unit tests, release lint, release build, APK signing, and 16 KB
  alignment.
- Physical-device install and launch with the signed release APK.
- Snapmaker U1 multi-nozzle G-code audit against the Orca reference fixture,
  including filament mapping, filament colors, nozzle metadata, toolchanges,
  prime tower geometry, and thumbnails.

## Beta Notes

- Review generated G-code before running long or unattended prints.
- Bambu LAN printing remains beta and may vary by printer model, firmware, and
  local network.
- Thingiverse account sign-in and direct import may depend on backend
  availability during the beta.
- Report install issues, crashes, printer connection problems, model import
  problems, and incorrect multicolor output through the project feedback
  channels.
