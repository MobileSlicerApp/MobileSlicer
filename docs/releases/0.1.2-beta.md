# MobileSlicer 0.1.2 Beta

MobileSlicer 0.1.2 Beta is a focused Android update for multi-material and
multi-nozzle workflows. This release restores Snapmaker U1 multi-nozzle color
output, improves Orca 3MF project handling, and ships as a newly versioned APK
so existing beta users can install it as a normal update.

This is the recommended beta build for Android testers.

## Highlights

- Restores multi-nozzle color output for Snapmaker U1 projects.
- Preserves Orca 3MF project material, color, and extruder assignments through
  the slicing path.
- Keeps STL imports single-material unless the user assigns colors in
  MobileSlicer.
- Maintains Snapmaker U1 prime tower behavior from the Orca profile: the tower
  is enabled, while purge-into-tower remains disabled for this printer profile.
- Adds a Snapmaker U1 Orca reference fixture for release checks.

## Install

Download `MobileSlicer-0.1.2-beta-v5.apk` from the assets below, then open it on
your Android device. Android may ask you to allow installs from your browser or
file manager before continuing.

If you already have an older MobileSlicer beta installed, this build should
install as an update.

## Verified

- Android unit tests, release lint, release build, APK signing, and 16 KB
  alignment.
- Physical-device install and launch with the signed release APK.
- Snapmaker U1 multi-nozzle G-code audit against the Orca reference fixture,
  including filament mapping, toolchanges, prime tower geometry, and thumbnails.

## Beta Notes

- Review generated G-code before running long or unattended prints.
- Bambu LAN printing remains beta and may vary by printer model, firmware, and
  local network.
- Thingiverse account sign-in and direct import may depend on backend
  availability during the beta.
- Report install issues, crashes, printer connection problems, model import
  problems, and incorrect multicolor output through the project feedback
  channels.
